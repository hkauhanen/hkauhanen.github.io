1. Intro

HTCondor (Condor for short) is wonderful for running batch jobs whenever individual parallel instances of the batch do not need to communicate with each other — this can include Monte Carlo simulations, systematic sweeps over the parameter space of a dynamical system, and many kinds of complicated data analysis tasks. It is also ingenious, in the sense that it may be used to harness the dormant power of ordinary compute nodes when they are not otherwise in use (e.g. university cluster computers at night and over weekends). A significant challenge arises from the use of such periodically available compute nodes, however — if the node ceases to be available before the job has run to completion, the job is lost and needs to be restarted from scratch when the node becomes available again.

Checkpointing is the procedure of saving the job’s state periodically so that if the job is evicted by the Condor queueing system, it can be resumed from where it left off, thus saving valuable CPU hours. Checkpointing can be implemented in many ways. Here, I describe how to implement user-level checkpointing for R jobs — something that seems not to have received much attention from Condor users, perhaps because few people use R for tasks that potentially last for many hours or days. (This makes sense, of course: if you can reasonably expect your simulations to be very computationally intensive, you’d probably be better off writing them in a compiled language to begin with. But sometimes this incurs a significant overhead in the form of having to write an amount of code that would be available off the shelf in R. So applications exist where deploying R for long-running jobs is genuinely sensible.)

2. Usage

The function I’ve written (download here or scroll down for the source) goes by the unimaginative name iterate_with_checkpointing. It is a simple wrapper that calls the desired routine for a set number of iterations, checkpointing at each step using R’s native capacity to save and load objects in binary. The function takes a number of arguments:

iterate_with_checkpointing(FUN, pars = list(), initial_value = list(), colnames, pass = "iterator_only", max_iteration, start_iteration = 1, checkpoint_file = "./checkpoint.RData")
  • FUN: the routine called at each iteration. This should be an R function that returns the result of one iteration to the wrapper routine.
  • pars = list(): a list of parameters to be passed to FUN. An empty list by default.
  • initial_value = list(): an initial state for the simulation. Useful if, for example, a dynamical system needs to be iterated from an initial condition. Empty by default.
  • colnames: a character vector of column names for the resulting data frame.
  • pass = "iterator_only": determines the behaviour of the wrapper: what exactly is passed on to FUN? This can be one of the following character strings (or any unique substring of them): "iterator_only", "previous_iteration", "full_results".
    • "iterator_only": FUN only receives information about the current iteration number; this option is useful for Monte Carlo runs where one repetition is not dependent on the previous one.
    • "previous_iteration": useful for iterating a dynamical system, as the results of the \((i-1)\)th iteration are passed to FUN for calculating the \(i\)th iteration.
    • "full_results": this option makes all iterations \(j\) with \(j < i\) available to FUN when calculating the \(i\)th iteration.
  • max_iteration: index of the final iteration step.
  • start_iteration = 1: which integer should the loop start at. Default is 1, so one gets max_iteration iterations overall.
  • checkpoint_file = "./checkpoint.RData"): temporary file used for checkpointing. Make sure this file does not exist before executing the routine.

4. Source code

iterate_with_checkpointing <- function(FUN,
                                       pars = list(),
                                       initial_value = list(),
                                       outcolumns,
                                       pass = "iterator_only",
                                       max_iteration,
                                       start_iteration = 1,
                                       checkpoint_file = "./checkpoint.RData") {
  # check whether checkpoint file exists; if so, load environment
  if (file.exists(checkpoint_file)) {
    load(checkpoint_file)
  } else {
    # if checkpoint file does not exist, we initialize a data frame for the
    # results and set checkpoint iterator to the start value. If user has
    # provided an initial value, this is taken into account
    if (length(initial_value) == 0) {
      checkpointing_iter <- start_iteration
      results <- data.frame(iteration=1:max_iteration, stringsAsFactors=FALSE)
      for (oc in outcolumns) {
        results[[oc]] <- NA
      }
    } else {
      checkpointing_iter <- start_iteration + 1
      results <- data.frame(iteration=1:max_iteration, stringsAsFactors=FALSE)
      for (oc in outcolumns) {
        results[[oc]] <- NA
        results[[oc]][1] <- initial_value[[oc]]
      }
    }
  }

  # check what sort of pass behaviour was requested
  possible_behaviours <- c("iterator_only", "previous_iteration", "full_results")
  to_pass <- charmatch(pass, c("iterator_only", "previous_iteration", "full_results")) # this is 1, 2 or 3 depending on selection

  # iterate
  if (is.na(to_pass)) {
    stop("Invalid value of 'pass' argument. Possible values are 'iterator_only', 'previous_iteration' and 'full_results', or any of their partial matches.")
  } else if (to_pass == 1) {
    while (checkpointing_iter <= max_iteration) {
      # add desired stuff to pass to FUN
      pars$iteration = checkpointing_iter

      # call FUN
      results[checkpointing_iter, ] <- do.call(FUN, pars)

      # increase iteration, save current state
      checkpointing_iter <- checkpointing_iter + 1
      save(results, checkpointing_iter, file=checkpoint_file)
    }
  } else if (to_pass == 2) {
    while (checkpointing_iter <= max_iteration) {
      # add desired stuff to pass to FUN
      pars$iteration = checkpointing_iter
      pars$previous_iteration = results[checkpointing_iter - 1, ]

      # call FUN
      results[checkpointing_iter, ] <- do.call(FUN, pars)

      # increase iteration, save current state
      checkpointing_iter <- checkpointing_iter + 1
      save(results, checkpointing_iter, file=checkpoint_file)
    }
  } else if (to_pass == 3) {
    while (checkpointing_iter <= max_iteration) {
      # add desired stuff to pass to FUN
      pars$iteration = checkpointing_iter
      pars$previous_iteration = results[checkpointing_iter - 1, ]
      pars$full_results = results

      # call FUN
      results[checkpointing_iter, ] <- do.call(FUN, pars)

      # increase iteration, save current state
      checkpointing_iter <- checkpointing_iter + 1
      save(results, checkpointing_iter, file=checkpoint_file)
    }
  }

  # return
  results
}
